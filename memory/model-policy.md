# 模型选择策略 - 检索卡片

> 此文件用于检索优化，保持简洁关键词风格

## 模型别名

| 别名 | 实际模型 |
|------|----------|
| GLM | zai/glm-5 |
| gpt54 | openai-codex/gpt-5.4 |
| minimax | bailian/MiniMax-M2.5 |
| sonnet | anthropic/claude-sonnet-4-6 |

## 选择策略

### 优先级
`coding > complex reasoning > simple task > default`

### 场景对照表

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 主会话日常对话 | glm-5 | 默认，性价比高 |
| 复杂分析/深度推理 | gpt-5.4 | 推理能力强 |
| 代码编写/重构/review | gpt-5.4 | coding 能力强 |
| 摘要/改写/翻译 | MiniMax-M2.5 | 简单任务，省钱 |
| Sonnet 场景 | 用 gpt-5.4 替代 | 除非明确指定 |

## Token 节省模式

- **保质量优先的节省 token 模式**
- 该省的省：冗余读写、重复调用、无效往返
- 不该省的不省：关键推理、关键校验、关键步骤
- **确保结果质量不下降**

## 子代理模型选择

- 复杂任务 → 派 gpt-5.4 子代理
- 简单任务 → 派 MiniMax 子代理
- 默认 → 不指定，用主会话模型
