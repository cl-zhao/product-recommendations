## 4. M1 Mac 必须知道的事情

### 4.1 Rosetta 2：运行未适配 M1 的 Intel 软件

**Rosetta 2 是苹果出的"翻译层"，让 x86 软件能在 ARM Mac 上运行。**

**使用步骤：**

1. 首次打开 Intel 软件时，系统会弹出提示：
   > "xxx.app 需要安装 Rosetta 才能打开。点击"安装"继续。"
2. 点击 **"安装"**，输入密码确认
3. 等待安装完成（几秒钟）

**或手动安装：**
```bash
# 在终端运行
softwareupdate --install-rosetta
```

> ⚡ **性能损失**：Rosetta 转译后软件性能大约下降 10-20%，大部分日常使用感知不到，但专业软件（如某些 PS 插件）可能会有卡顿。

---

### 4.2 原生 ARM 软件 vs Intel 软件：如何判断

**方法 1：查看应用信息**

1. 打开 **访达 (Finder)**
2. 进入 `/Applications`
3. 右键点击应用 → **"显示简介" (Get Info)**
4. 查找 **"种类" (Kind)**：
   - ✅ "Apple Silicon" = 原生 ARM
   - ❌ "Intel" = 需要 Rosetta 转译

**方法 2：活动监视器**

1. 打开 **活动监视器 (Activity Monitor)**
2. 点击 **"CPU"** 标签
3. 查看 **"种类" (Kind)** 列：
   - ✅ Apple = 原生 ARM
   - ❌ Intel = 转译运行

---

### 4.3 Homebrew 安装：x86 还是 ARM？

**Homebrew 支持双版本：**

| 版本 | 安装路径 | 使用场景 |
|------|----------|---------|
| **ARM 版（推荐）** | `/opt/homebrew` | 原生 ARM，速度快 |
| **x86 版** | `/usr/local` | 需要跑旧 Intel 软件 |

**安装步骤（ARM 版，推荐）：**

```bash
# 安装 ARM 版（新版默认）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 验证版本
brew --version
# 输出类似: Homebrew 4.x.x (ARM)
```

**安装 x86 版（如必须）：**

```bash
# 在终端运行
arch -x86_64 /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

> 💡 **建议**：除非有特殊需求（如旧的 x86 依赖），否则用 ARM 版。两者可以共存，但软件包不共享。

---

### 4.4 Docker Desktop：M1 版和 Intel 版区别

**下载链接：** https://www.docker.com/products/docker-desktop/

**选择版本：**
- ✅ **Apple Silicon** = M1/M2/M3 Mac 原生
- ❌ ~~Intel 版本~~ = 不能在 M1 上用

**安装注意事项：**

1. **镜像兼容性**：
   - 官方镜像（如 `nginx`、`mysql`）基本都有 `arm64v8` 版本
   - 某些第三方镜像可能只有 x86 版本，需要 `--platform linux/amd64` 强制运行

2. **性能表现**：
   - Docker 在 M1 上运行 Linux 容器**原生**（不需要完整虚拟机），性能接近原生
   - 但 x86 容器通过 QEMU 模拟，性能损失约 10-30%

3. **常见问题**：
   - 如果容器启动报 `exec format error`，通常是镜像不支持 ARM64
   - 检查镜像是否有 `arm64` 或 `linux/arm64/v8` 标签

---

### 4.5 虚拟机：Parallels / UTM 跑 Windows ARM 版

**两个选择：**

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| **Parallels Desktop** | 付费（¥368/年），简单易用 | 偶尔跑 Windows 软件 |
| **UTM** | 免费，开源 | 轻度使用 or 预算有限 |

**Windows ARM 版下载：**
- 官方预览版：https://www.microsoft.com/en-us/windowsinsider/download
- 需要加入 Windows Insider 计划

**Parallels 使用步骤：**

1. 下载安装 Parallels Desktop
2. 启动 → 选择 "安装 Windows"
3. 选择 "下载 Windows ARM 预览版"（会自动下载）
4. 按照提示完成安装
5. 安装后运行 Windows Update 更新

**UTM 使用步骤：**

1. 下载 UTM：https://getutm.app/
2. 下载 Windows ARM VHDX：https://aka.ms/arm64vhd
3. 打开 UTM → 新建虚拟机
4. 选择 "Windows"
5. 加载 VHDX 文件
6. 启动虚拟机

> ⚠️ **注意**：Windows ARM 版只能运行 **ARM 原生应用**（如 Edge、Office 365）。传统 x86 Windows 软件需要 Parallels 的转译层（免费）或 UTM 的 QEMU 模拟（慢）。

## 5. 温度和续航验证

### 5.1 正常温度范围

**查看温度方法：**

- **简单方法**：无需安装任何软件，直接看
- **精确方法**：下载 ** Macs Fan Control** 或 **TG Pro**（付费）

| 状态 | 正常温度 |
|------|----------|
| **空闲/日常办公** | 35°C - 45°C |
| **浏览网页/文档** | 40°C - 55°C |
| **轻度负载（看视频）** | 50°C - 65°C |
| **满载（渲染/跑分）** | 80°C - 100°C |
| **充电时** | 稍高 5-10°C 正常 |

> 🌡️ M1 Pro 最高可以跑到 100°C 不降频，苹果设计了过热保护。偶尔跑到 90-100°C 不用慌，持续高温才需要关注。

---

### 5.2 风扇噪音正常水平

| 状态 | 风扇转速 | 噪音水平 |
|------|----------|---------|
| **空闲/轻度** | 停转或 1000 RPM | **0 dB（静音）** |
| **中等负载** | 2000-3000 RPM | ~25-30 dB（几乎听不见） |
| **满载** | 5000-6000 RPM | ~35-40 dB（能听到但不吵） |

> 🔇 **M1 Pro 的风扇非常安静**，日常使用基本听不到。只有长时间渲染、跑分时才会听到风扇声音。

**快速检查风扇是否正常：**

```bash
# 在终端运行
sudo powermetrics --samplers smc | grep -E "Fan|Fan Speed"
```

**官方参考数据：**

> 🔋 实际续航取决于你的使用场景。如果开很多 Chrome 标签页 + 外接显示器，续航会明显下降。

## 6. 常见 M1 Pro 问题排查

### 6.1 风扇狂转：可能原因

**正常情况：**
- ✅ 正在跑大型软件（视频导出、编译、渲染）
- ✅ 系统更新中
- ✅ 充电时高负载

**异常情况排查：**

1. **检查活动监视器**：
   - 打开活动监视器 → 按 CPU 排序
   - 看看哪个进程占用高
   - 如果是 `kernel_task`，说明系统在工作，可能是 Spotlight 索引

2. **重置 SMC**（相当于 BIOS 重置）：
   - 关机
   - 按住 **Control + Option + Shift** + **电源键** 10 秒
   - 等待 30 秒，再开机

3. **检查是否有恶意软件**：
   - 使用 **Malwarebytes for Mac**（免费）全盘扫描

4. **如果持续狂转且温度不高**：
   - 可能是风扇传感器故障，建议天才吧检测

---

### 6.2 发热异常：可能原因

**排查步骤：**

1. **检查是否在充电**：充电时发热正常
2. **降低屏幕亮度**：高亮度增加功耗和发热
3. **关闭不用的软件**：特别是 Chrome（CPU 占用高）
4. **检查后台进程**：
   - 活动监视器 → CPU → 排序找高占用
5. **如果是特定软件发热**：
   - 尝试更新到最新版本
   - 或查找替代软件

**如果温度持续 > 90°C 且风扇不转：**
- 可能是风扇故障，联系 Apple 支持

---

### 6.3 Wi-Fi 断连：解决方法

**常见原因和解决方案：**

1. **重置网络设置**：
   - 系统设置 → 网络 → 点击左下角 **管理隐藏网络**
   - 删除所有保存的网络，重新连接

2. **重置 SMC**（同上）

3. **更新系统**：确保 macOS 是最新版本

4. **检查路由器**：尝试改 Wi-Fi 频道（避开 2.4G 拥挤信道）

5. **关闭蓝牙**：
   - 有些用户反馈蓝牙和 Wi-Fi 干扰
   - 系统设置 → 蓝牙 → 关闭

6. **忘记网络重新连接**：
   - 系统设置 → Wi-Fi → 点击 **"i"** → **"忽略此网络"**
   - 重新连接

---

### 6.4 USB-C 不识别设备：排查步骤

1. **换个端口试试**：有些端口可能供电不足
2. **换根线**：有些线材质量差
3. **检查设备供电**：移动硬盘需要独立供电
4. **重启 Mac**：有时候系统识别会卡住
5. **重置 SMC**（见上文）
6. **检查系统报告**：
   - 苹果菜单 → 关于本机 → 系统报告 → USB
   - 查看设备是否被识别
7. **更新 macOS**：某些外设兼容性问题会在更新中修复
