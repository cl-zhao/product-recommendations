### 15. @levie

**Aaron Levie** · 04-30 21:15

当 agents 成为软件的最大用户时，那么所有软件都必须以无头（headless）方式提供。agents 不会使用你的 UI，他们会直接调用你的 API。

所以问题变成了：软件的商业模式在这种无头方式下将何去何从？

以下是我基于在 Box 所见所做的一些思考，以及与其他平台的交流：

1. **人的席位不会消失。** 席位仍然是一种便捷高效的方式，让客户为一组用户在基准使用量范围内可预测地使用技术。但关键在于，当客户为席位付费时，必须包含该席位对应的 API 使用量，由 agent 代表该用户调用。

用户需要能够通过他们使用的任何 agent 与他们的数据和底层工具进行交互，席位将附带相对较高容量的 API 使用量。我预计大多数软件——Box 在内——将让席位通过 ChatGPT、Codex、Claude、Gemini、Cursor、Copilot、Perplexity、Factory、Cogniton 等系统与用户数据无缝协作。如果你不这样做，你就死定了（DOA）。

2. **Agents 可能有"席位"**，如果它们在系统中做有状态的工作，但定价将与人大不相同。当 agent 有自己的workspace、存储自己的数据、需要与用户不同的权限集时，席位（或等价物）就有意义了。

如果一家公司希望这个 agent 长期存在，这很可能看起来像是系统中的另一个"用户"。OpenClaw 风格的 agents 展示了一种可能的未来图景。

这里唯一的问题是定价：一个客户可能决定用 1 个 agent 完成所有工作，另一个可能将其拆分为 1000 个 agents。所以像人类席位那样定价几乎不可能也不实际——每家公司都会有不同的方法。

3. **无头使用的 dominant 定价模式** 是消耗模型，超出席位配额的部分，或者当 agent 坚定地自主行动时。许多企业软件平台此前通过 PaaS 选项运营，agents 将成为其系统的另一个机器用户。

在某些情况下，API 可能按照以前的方式定价，但在其他情况下，可能需要新型 API 来代表 agent 一次性完成的工作——更像是结果导向而非一系列 API 调用。当无头软件本身也嵌入了 agentic 用例时，这一点尤为重要——比如在其系统内通过 AI 编排工作流程。

总体而言，这种使用模式的增长实际上是无限 unbounded 的，因为 agents 在这些系统中对数据的操作用例将大大超过人们今天对自己的数据和工具所做的操作。每个走向无头化的平台（也就是任何想利用 agents 优势的）都需要采用这样的模式。一些人最初可能会抵制，但这是必然趋势——因为你的平台外部的 agents 永远会比人更多。

总的来说，由于这些系统的无头使用，软件的未来还有非常多有趣的变化。才刚开始。

### 18. @ryolu_

**Ryo Lu** · 04-30 14:58

没有偏好本身也是一种偏好 / no opinion is opinionated

人们把"有主见"（opinionated）和"通用目的"（general purpose）当作反义词。并不是。

你决定什么概念存在、什么可见、什么可组合、什么可扩展、什么隐藏的那一刻——你已经表达了一种世界观。"没有偏好"本身就是一种强烈的偏好。

为普遍性而构建意味着说一个非常具体的事情：核心概念应该少而持久，且灵活。用户不应被困在僵化的工作流中。抽象比表面专业化更重要。系统应该能折叠成多种形态而不破坏其底层逻辑。

这也是为什么纯粹的"客户至上"是不够的。用户反馈告诉你摩擦在哪里。它很少告诉你底层系统应该是什么。如果你只遵循直接的用户需求，你会收敛到局部最优——碎片化的功能、狭窄的工具、围绕当前习惯而非持久概念塑造的产品。

那个差距就是设计存在的地方。简化现实到显而易见的水平以下的工作。

伟大的系统不是从做加法开始的。它们从把概念精简到本质开始——直到原语变得清晰。然后为每个概念的扩展、组合、折叠到其他概念构建路径。包装可以变化。本质不应该。

这正是让系统能够服务初学者而不限制专家、服务一个领域而不被其困住的原因。

灵活性不是没有偏好。是有内聚力的强烈偏好——不是关于一个理想工作流，而是关于哪些概念是基础的、哪些抽象应该持久、哪些自由度重要。

核心简单。边缘开放。这就是终极主张。
