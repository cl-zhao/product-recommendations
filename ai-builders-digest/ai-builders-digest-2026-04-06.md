### 12. @levie

**Aaron Levie** · 04-04 20:35

As AI models get better at handling tools, and as context windows get bigger without as much rot, you can start to design agents more similar to how people work instead of having to mitigate the model limitations with weird hacks.
随着 AI 模型在工具调用上越来越强，上下文窗口越来越大且衰减减少，你可以开始把 Agent 设计得更像人一样工作，而不必用各种奇怪的 hack 来弥补模型的局限性。

For instance, even a year ago, if you were to build an agent to process large amounts of documents, the state of the art was to do embeddings on the data, then do a similarity search and pull out the chunks of content that matched (as well as surrounding chunks). This was necessary because context windows could only accurately handle a small amount of information at a time.
举个例子，即使在一年前，如果你要构建一个处理大量文档的 Agent，最先进的方法是对数据做 embedding，然后做相似度搜索，提取匹配的内容块（以及周围的内容块）。这是必要的，因为上下文窗口一次只能准确处理少量信息。

This worked surprisingly well given the constraints (at least assuming you were working with authoritative data only), but had a lot of tricky limitations because it's not how humans work.
在约束条件下这种方法效果出奇地好（至少在只处理权威数据的情况下），但有很多棘手的局限性，因为它不符合人类的工作方式。

For instance, what do you do if the chunks you sent to the model were the most relevant semantically, but actually rendered irrelevant by some other part of the document.
比如，如果你发给模型的内容块在语义上最相关，但实际上被文件的另一部分否定掉了，该怎么办？

For instance, if at the top of the document it says "do not use this" but on page 3 there is information that's relevant, that data will be sent to the model as the top hit.
比如，如果文件顶部写着"不要使用这个"，但第 3 页有相关信息，那个数据会作为最高匹配被发送给模型。

Similarly, chunked data is difficult when you need various parts of a document or many documents to be understood for answering a problem.
同样，当你需要理解文档的各个部分或多个文档来回答问题时，分块数据就很困难。

Today, increasingly, you can begin to have agents effectively use tools and work with information far more similar to how people work. This unlocks a qualitatively different set of use-cases and capability level that agents can now handle.
如今，越来越多地，你可以让 Agent 像人一样有效地使用工具和处理信息。这解锁了一套全新的用例和能力水平。

As we were designing the Box Agent, these improvements allowed us to rethink our entire architecture for AI.
在设计 Box Agent 时，这些改进让我们得以重新思考整个 AI 架构。

The agent can now search data similar to how a user searches, but with the benefit of being able to expand their queries, do semantic search, and process results nearly instantly.
Agent 现在可以像用户一样搜索数据，但能够扩展查询词、做语义搜索、近乎即时地处理结果。

Then the agent can either read many documents at a time or at least much larger amounts of context. Again, much more similar to people, but now at hyperspeed.
然后 Agent 可以一次读取许多文档，或者至少是大量得多的上下文。同样，更像人了，但现在是超音速。

Importantly, beyond tool calling and context windows, the reasoning of models has also gone up enormously. This means the agent can also know when it needs to search for information again when it didn't find something it was looking for or if something feels off.
重要的是，除了工具调用和上下文窗口，模型的推理能力也大幅提升了。这意味着 Agent 也能知道自己什么时候需要再次搜索信息——当它没找到想要的东西或感觉哪里不对的时候。

As model progress continues on the dimensions of context accuracy, tool calling, advanced reasoning, and coding, agents are going to become insanely powerful.
随着模型在上下文准确性、工具调用、高级推理和编程这些维度上持续进步，Agent 将变得极其强大。
